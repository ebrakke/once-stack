ONCE Stack UI Exploration

Shared UI elements for independently deployed ONCE apps.

The current notes app is clean and functional: centered content, white cards, gray background, blue primary actions. These prototypes explore how to make each separately deployed app — Notes, Files, Blog — feel like it belongs to the same family without presenting them as one combined shell.

Use the tabs and state buttons to click through list, view, form, and empty screens.

Palette · Calm Minimal

Option A · Calm Minimal

Best near-term evolution: keeps the existing simple centered layout but makes it warmer, more polished, and common across apps.


Option B · Single-App Sidebar

A sidebar can still work, but only for navigation inside one deployed app — for example folders/tags in Notes or directories in Files. It should not imply all ONCE apps are bundled together.


Option C · Editorial Warm

Best for notes/blog as reading spaces. It swaps the app feel for a paper-like, document-first experience with serif type and terra cotta accents.


Shared design guidelines

Layout

  • Each deployable app gets its own header and identity: Notes, Files, or Blog.
  • Share page width, spacing, card, button, form, and markdown tokens across apps.
  • Use sidebars only for in-app concepts like tags, folders, or drafts — not cross-app navigation.
  • Keep touch targets at least 40–44px high.

Components

  • Standardize .once-btn, cards, empty states, alerts, fields, markdown content.
  • Every app should have list, detail, form, empty, error states.
  • Prefer semantic tokens over app-specific colors.

Recommendation

  • Adopt Option A tokens first: lowest migration cost and best fit for independent apps.
  • Use Option B only as an in-app sidebar pattern, not a global ONCE shell.
  • Borrow Option C typography for Blog/post views.
